TLDR; (too long didn‚Äôt read)

Encryption = lock & unlock with a key! üîê

  • Encryption can be reversed - you can decrypt messages back to the original
  • You need a key to encrypt AND decrypt
  • Without the key, encrypted messages look like gibberish
  • This protects messages while they‚Äôre being sent (like emails or texts)
  • Different from hashing: encryption is two-way, hashing is one-way only

Background Information

Remember Alice and Bob from the book cipher? They want to send secret messages to each other, but Eve (the eavesdropper) is trying to read them!

Encryption is like putting your message in a locked box. Only someone with the right key can unlock it and read the message.

How does encryption work?

  1. Encrypt: Take your original message (called ‚Äúplaintext‚Äù) and scramble it using a key
  2. Send: Send the scrambled message (called ‚Äúciphertext‚Äù) - even if Eve intercepts it, she can‚Äôt read it!
  3. Decrypt: The receiver uses the same key to unscramble the message back to plaintext

Why is this important?

Every day, encryption protects your data: - üîí HTTPS websites encrypt your browsing so hackers can‚Äôt see what you‚Äôre doing - üí¨ Messaging apps encrypt your messages so only you and the recipient can read them - üí≥ Online shopping encrypts your payment details so they can‚Äôt be stolen - üìß Email can be encrypted to keep your private conversations safe

Encryption vs Hashing

Feature Encryption üîê Hashing üîí
Can it be reversed? ‚úÖ Yes (with the key) ‚ùå No (one-way only)
What‚Äôs it used for? Protecting data in transit Protecting stored passwords
Do you need a key? ‚úÖ Yes (to encrypt AND decrypt) ‚ùå No (just hash it)
Example Encrypted email message Hashed password in database

Both are important for security, but they solve different problems!

What is a Caesar Cipher?

The Caesar cipher is one of the oldest encryption methods! It was used by Julius Caesar over 2000 years ago. It works by ‚Äúshifting‚Äù each letter in the alphabet by a certain number of positions.

Example with shift of 3: - A ‚Üí D (A is the 1st letter, shift 3 ‚Üí D is the 4th letter) - B ‚Üí E - C ‚Üí F - ‚Ä¶ - X ‚Üí A (wraps around to the beginning) - Y ‚Üí B - Z ‚Üí C

The shift number is the key!

Step 1: Set up your encryption tool

Run this Python code to create your Caesar cipher encryption and decryption tool:

Code 
def caesar_encrypt(message, shift):
    """Encrypt a message using Caesar cipher with a given shift (key)"""
    encrypted = ""
    
    for char in message:
        if char.isalpha():  # Only shift letters, not spaces or punctuation
            # Determine if it's uppercase or lowercase
            if char.isupper():
                # Shift uppercase letters
                shifted = chr((ord(char) - ord('A') + shift) % 26 + ord('A'))
            else:
                # Shift lowercase letters
                shifted = chr((ord(char) - ord('a') + shift) % 26 + ord('a'))
            encrypted += shifted
        else:
            # Keep spaces, punctuation, and numbers as they are
            encrypted += char
    
    return encrypted


def caesar_decrypt(encrypted_message, shift):
    """Decrypt a message using Caesar cipher with a given shift (key)"""
    # Decrypting is just encrypting with the opposite shift!
    return caesar_encrypt(encrypted_message, -shift)

import hashlib

message = "secret"
encryption_key = 5

# Encrypt it
encrypted = caesar_encrypt(message, encryption_key)

print(f"\nHASHING (one-way only):")
print(f"  Hashed:    {hashed}")
print(f"  Reversed:  ‚ùå IMPOSSIBLE - can't get original back!")
print(f"\nKey difference: Encryption = two-way, Hashing = one-way")

Discussion Questions

After completing the investigations, discuss these questions:

  1. Two-way process: Why can encryption be reversed, but hashing cannot?

  2. The key: Why is the key so important? What happens if someone intercepts your encrypted message but doesn‚Äôt have the key?

  3. Key security: If Eve steals your encrypted message, can she figure out the key by trying different numbers? (This is called a ‚Äúbrute force attack‚Äù)

  4. Real world: When would you use encryption instead of hashing? When would you use hashing instead of encryption?

  5. Caesar cipher weakness: The Caesar cipher is easy to break. Can you think of why? How do modern encryption methods protect against this?

  6. Alice and Bob: How does encryption help Alice and Bob communicate privately, even if Eve is listening?

Key Takeaways

‚úÖ Encryption is two-way - You can encrypt AND decrypt messages

‚úÖ Keys are essential - You need the same key to encrypt and decrypt

‚úÖ Protects data in transit - Encryption keeps messages safe while being sent

‚úÖ Different from hashing - Encryption is reversible, hashing is one-way only

‚úÖ Both are important - Encryption and hashing solve different security problems

‚úÖ Without the key = gibberish - Encrypted messages are useless without the correct key
